<Accordion
  question="What is Dinah?"
  answer="Dinah is a thin, type-safe TypeScript client for DynamoDB. It wraps the AWS SDK v3 DocumentClient with a Repo abstraction whose return types are inferred from a typebox schema and your table definition."
  type="basic"
/>
<Accordion
  question="Do I need typebox?"
  answer="Yes — Dinah uses typebox schemas as the single source of truth for the shape of items in a table. typebox is listed as an optional peer dependency."
  type="basic"
/>
<Accordion
  question="Does Dinah work with local DynamoDB?"
  answer="Yes. Pass any DynamoDBDocumentClient — including one pointed at a local DynamoDB instance — when constructing the Db. Dinah does not assume cloud-only behavior."
  type="basic"
/>

<Accordion
  question="How do I define a GSI?"
  answer="Pass a `gsis` map in your Table definition. Each entry specifies a partitionKey, an optional sortKey, and an optional projection (ALL by default, KEYS_ONLY, or an explicit list of attributes). Dinah infers query argument and return types from this."
  type="setting"
/>
<Accordion
  question="Why does my GSI query return a partial item?"
  answer="If the GSI has a KEYS_ONLY or INCLUDE projection, Dinah narrows the return type to exactly the attributes that DynamoDB will actually return — the table keys plus any projected attributes."
  type="setting"
/>
<Accordion
  question="Can I add computed fields to items?"
  answer="Yes — subclass `AbstractRepo` and override `transformItem`. Dinah threads the transformed type through every read operation that returns full items."
  type="setting"
/>

<Accordion
  question="Where do I report bugs?"
  answer="Open an issue on GitHub at github.com/jayalfredprufrock/dinah/issues. PRs welcome too."

/>
<Accordion
